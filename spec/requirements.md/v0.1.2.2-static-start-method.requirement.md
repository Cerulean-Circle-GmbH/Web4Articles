# Static Start Method Requirement v0.1.2.2

**🗓️ Date:** 2025-09-06-UTC-2115  
**🎯 Requirement ID:** v0.1.2.2-static-start-method  
**🎯 Version:** 0.1.2.2  

## Requirement Statement

**All CLI components MUST implement static start method for radical OOP compliance - no main functional entity point, only static start method receiving shell parameters.**

## Context

From TRON Requirements: "starting in radical oop always means, no main functional entity point but a static start method getting the sh parameters."

## Specification

### **Static Start Method Pattern:**
```typescript
// CORRECT - Radical OOP Static Start
class ComponentCLI {
  static async start(args: string[] = []): Promise<void> {
    const cli = new ComponentCLI();
    await cli.run(args);
  }
}

// CLI execution
ComponentCLI.start(process.argv.slice(2));
```

### **Prohibited Patterns:**
```typescript
// INCORRECT - Main functional entity point
function main() {
  const cli = new ComponentCLI();
  cli.run();
}

// INCORRECT - Direct instantiation at module level
const cli = new ComponentCLI();
cli.run();
```

### **Shell Parameter Handling:**
- **Static Method:** Receives shell parameters directly
- **No Main Function:** No main functional entity point
- **Instance Creation:** Static method creates instance internally
- **Parameter Passing:** Shell parameters passed to instance methods

### **npm start Integration:**
```json
{
  "scripts": {
    "start": "node dist/ts/layer5/ComponentCLI.js"
  }
}
```

### **CLI File Implementation:**
```typescript
#!/usr/bin/env node

import { ComponentCLI } from './ComponentCLI.js';

// Radical OOP: Static start method with shell parameters
ComponentCLI.start(process.argv.slice(2)).catch(error => {
  console.error('Fatal error:', error);
  process.exit(1);
});
```
